    Makefile学习

    Makefile文件基本规则

    目标 : 依赖 tab键 命令

    • 目标: 所要生成的目标
    • 依赖: 生成目标需要的依赖文件
    • 命令: 依赖生成目标的执行命令

    总结: - Makefile并不会关心命令是如何执行的，仅仅只是回去执行所定义的命令，和我们平时直接输入命令行是一样的效果 - Makefile就相当于一个依赖文件关系，在执行该文件以后会递归的查找依赖文件并执行对应的命令，最终生成第一个目标。

    简单使用Makefile

    Test : Test.o module.o module2.o
    gcc Test.o module1.o module2.o -o Test
Test.o : Test.c
    gcc -c Test.c -o Test.o
module1.o : module1.c
    gcc -c module1.c -o module1.o
module2.o : module2.c
    gcc -c module2.c -o module2.o

    上图是makefile文件，最终为了生成Test文件，需要三个.o文件，然后三个.o文件又分别依赖于对应的.c文件，这样加上对应的gcc命令就构成了一个基本的makefile。

    使用指定Makefile文件

    在命令行中使用make命令，makefile会默认路径中查找对应的makefile文件来进行工程管理，我们一般把对应文件命名为Makefile和makefile，如果名字不匹配可能无法找到对应的makefile文件。 不过如果平时在一个目录下存在多个makefile文件的时候，一般我们都会通过不同命名来进行区分，那么该如何指定对应的makefile文件进行识别解析呢？可以使用make -f选项

    make -f

    注释文本

    在开发过程中需要注释掉makefile文件相应文本行可以在文件的行首使用#号。

    取消回显文本

    在makefile使用过程中默认是会把相应的处理过程信息进行回显，这样能够让开发者更好的了解makefile的处理过程。 不过，如果回显信息非常的庞大也是不利于开发者分析的，所以可以使用@来取消相关信息的回显，如下所示:

    Test : Test.o module.o module2.o
    @gcc Test.o module1.o module2.o -o Test
Test.o : Test.c
    @gcc -c Test.c -o Test.o
module1.o : module1.c
    @gcc -c module1.c -o module1.o
module2.o : module2.c
    @gcc -c module2.c -o module2.o

    伪目标的使用

    在makefile文件中有时候一些不需要有依赖命令目标，称为伪目标。

    Test : Test.o module.o module2.o
    @gcc Test.o module1.o module2.o -o Test
Test.o : Test.c
    @gcc -c Test.c -o Test.o
module1.o : module1.c
    @gcc -c module1.c -o module1.o
module2.o : module2.c
    @gcc -c module2.c -o module2.o

.PHONY : clean
clean :
    rm *.o Test

    上面是对应的makefile文件，其中里面加入伪目标clean，通过使用.PHONY修饰，这样可以直接使用 make clean 来执行对应的命令。 上面执行make clean 执行了清楚操作。

    变量的使用

    在makefile中为了减少代码的重复，使用变量的概念来简化编写，如下面的makefile进行如下改造:

    obj = Test.o module1.o module2.o
Test : $(obj)
    @gcc $(obj) -o Test
Test.o : Test.c
    @gcc -c Test.c -o Test.o
module1.o : module1.c
    @gcc -c module1.c -o module1.o
module2.o : module2.c
    @gcc -c module2.c -o module2.o

.PHONY : clean
clean :
    rm *.o Test

    使用$(obj)来代替所有的目标文件，以后需要添加其他目标文件的话就只需要在变量obj出进行相应修改即可。 同时系统还存在其他默认的自动化变量，这样可以大大简化makefile文件，便于设计和后期维护，如: - $^ 表示所有的依赖文件 - $@ 表示生成的目标文件 - $< 表示第一个依赖文件

    make命令

    • -C DIRECTORY, --directory=DIRECTORY Change to DIRECTORY before doing anything. 可以为make命令提供不同的目录路径，在寻找Makefile之前会切换目录。 如果想运行make命令的Makefile保存在 ../test1/ 目录下，可以这样做: make -C ./test1

      make: Entering directory '/home/superman/for_make/test1' gcc -c test1.c -o test1.o make: Leaving directory '/home/superman/for_make/test1'

    可以看到make命令首先切换到特定的目录下执行makefile，然后再切换回来。
